# 竞拍之王计算器

一个用于《竞拍之王》猜总价场景的 React + Electron 计算器。它根据总藏品数、不同颜色品质的件数、均格和候选件数规则，帮助快速估算总价，并在信息不足时列出可能组合。

## 设计思路

游戏里的核心问题是：玩家通常能看到一批藏品的颜色品质，但很难直接知道每个品质的准确件数和总价。这个工具把估算拆成几层：

1. 按颜色品质录入信息：白/绿、蓝色、紫色、金色、红色。
2. 每个品质输入“确定件数”和“均格”，自动计算总格。
3. 根据均格的小数部分反推“可能件数”。例如 `1.33` 会使用 `0.33` 对应的候选件数。
4. 对特殊品质加上经验上限：紫色最多 20 件，金色最多 12 件，红色最多 5 件。
5. 给定总藏品数后，工具会用已知件数缩小候选范围。
6. 点击“一键推算”后，枚举未确定颜色的可能组合，筛选出满足总藏品数的结果，并估算每种组合的总价。

红色比较特殊：红色价格不看均格，总价按 `红色件数 * 100000` 计算。当前四种非红色品质都确定后，红色件数会按总藏品数自动推算。

## 价格规则

| 品质 | 总价计算 |
| --- | --- |
| 白/绿 | 总格 * 200 |
| 蓝色 | 总格 * 500 |
| 紫色 | 总格 * 3000 |
| 金色 | 总格 * 10000 |
| 红色 | 件数 * 100000 |

其中：

```text
总格 = 确定件数 * 均格
```

## 候选件数规则

候选件数表放在：

```text
src/averageRemainderCandidates.js
```

工具只使用均格的小数部分做推理：

```text
1.33 -> 查 0.33
2.70 -> 查 0.70
2.7  -> 查 0.7
2    -> 整数，认为 1-30 都可能
```

为了避免输入 `1.33` 时刚输到 `1.3` 就自动填值，自动填写“确定件数”只会在小数至少两位时触发。

## 网页版怎么用

在线访问：

```text
https://skyskyskyha.github.io/BidCalculator/
```

安装依赖：

```powershell
npm install
```

启动网页版开发环境：

```powershell
npm run dev
```

打开：

```text
http://localhost:5174
```

使用流程：

1. 输入总藏品数。
2. 输入各颜色的确定件数和均格。
3. 查看“可能件数”是否被规则缩小。
4. 如果某一行可能件数只剩一个，工具会自动填入确定件数。
5. 点击“一键推算”，查看最多 6 种可能组合和估算总价。

## 桌面 App 怎么用

打包 Windows 免安装版：

```powershell
npm run pack:desktop
```

生成目录：

```text
release-latest/win-unpacked
```

运行：

```text
release-latest/win-unpacked/竞拍之王计算器.exe
```

注意：不要只拷贝单个 exe。Electron 免安装版需要整个 `win-unpacked` 文件夹里的运行文件一起存在。

如果要发给别人，可以把整个 `win-unpacked` 文件夹压缩，或者使用已经生成的 zip：

```text
release-latest/竞拍之王计算器-latest-win-unpacked.zip
```
